## Base URL

All API requests are made to the following base URL:

```
https://kutt.it/api/v2
```

If you are self-hosting Kutt, replace the domain with your own:

```
https://yourdomain.com/api/v2
```

## Response format

All endpoints return JSON unless otherwise noted.

## Authentication

Every request must include your API key in the `X-API-KEY` header. See [Authentication](/api-reference/authentication) for details.

## Health check

You can verify that the API is reachable with a simple health check:

```bash
GET https://kutt.it/api/v2/health
```

A successful response returns the plain-text string `OK` with HTTP status `200`.

## Rate limiting

Rate limiting can be enabled on self-hosted instances by setting the `ENABLE_RATE_LIMIT` environment variable. When rate limiting is active and you exceed the allowed number of requests, the API returns an HTTP `429` response.
