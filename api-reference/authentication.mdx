## Getting your API key

1. Sign in to your Kutt account.
2. Go to **Settings**.
3. Under the API section, select **Generate API key**.
4. Copy the key and store it securely. Kutt does not display the key again after you leave the page.

## Using your API key

Pass your API key in the `X-API-KEY` header with every request.

```bash
curl https://kutt.it/api/v2/links \
  -H "X-API-KEY: your-api-key"
```

## Missing or invalid key

If you make a request without providing an API key, or if the key is invalid, the API returns:

```json
HTTP 401 Unauthorized
```

Ensure the header name is exactly `X-API-KEY` and that the value matches the key shown in your Settings page.

## Admin endpoints

Some query parameters and operations are restricted to admin accounts. To use them, authenticate with the API key belonging to an admin account. The [list links](/api-reference/links) endpoint, for example, accepts an `all` query parameter that is only available to admins.
