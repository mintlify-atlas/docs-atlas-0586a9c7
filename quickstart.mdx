## Prerequisites

- [Node.js](https://nodejs.org/) 20 or above
- Git

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the Kutt repository from GitHub:

    ```bash
    git clone https://github.com/thedevs-network/kutt.git
    cd kutt
    ```

    Alternatively, [download the latest release zip](https://github.com/thedevs-network/kutt/releases) and extract it.
  </Step>

  <Step title="Install dependencies">
    Install the required Node.js packages:

    ```bash
    npm install
    ```
  </Step>

  <Step title="Initialize the database">
    Run migrations to set up the database schema. Kutt uses SQLite by default — no external database required.

    ```bash
    npm run migrate
    ```
  </Step>

  <Step title="Create a .env file">
    Copy the example environment file and set your `JWT_SECRET`:

    ```bash
    cp .example.env .env
    ```

    Open `.env` and fill in `JWT_SECRET` with a long, random string:

    ```bash
    JWT_SECRET=your-long-random-secret-here
    ```

    <Note>
      `JWT_SECRET` is required in production. The app will use a default development value if it is not set, but you must provide a strong, unique value before exposing Kutt publicly.
    </Note>
  </Step>

  <Step title="Start the app">
    For development (with auto-reload on file changes):

    ```bash
    npm run dev
    ```

    For production:

    ```bash
    npm start
    ```

    Once started, you will see:

    ```
    > Ready on http://localhost:3000
    ```

    Open [http://localhost:3000](http://localhost:3000) in your browser.
  </Step>
</Steps>

## First start

When you open the app for the first time, you are prompted to create an admin account. Complete this step before using Kutt — the admin account gives you access to the admin panel where you can manage users and links.
