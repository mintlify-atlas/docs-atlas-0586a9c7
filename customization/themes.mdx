## CSS customization

Place your CSS files in `custom/css/`. Kutt loads them alongside its default styles, so you can override specific rules without replacing everything.

- You can add as many files as you want: `custom1.css`, `custom2.css`, and so on.
- Each file is served at `/css/<file>.css`.
- **Special case:** if you name a file `styles.css`, it replaces Kutt's original `styles.css` entirely instead of appending to it.

## Image customization

Place your images in `custom/images/`. To replace a built-in image, give your file the same name as the corresponding file in `static/images/`.

The built-in images are:

| File | Description |
|---|---|
| `logo.png` | Main logo |
| `favicon.ico` | Browser favicon |
| `favicon-16x16.png` | 16 Г— 16 favicon |
| `favicon-32x32.png` | 32 Г— 32 favicon |
| `favicon-196x196.png` | 196 Г— 196 favicon |
| `card.png` | Open Graph / social card image |

Each image is served at `/images/<image>.<format>`.

## View customization

Kutt uses [Handlebars](https://handlebarsjs.com/) (`.hbs`) templates for its HTML pages. To override a template, create a file in `custom/views/` that mirrors the path of the original file in `server/views/`.

For example, to override the footer partial:

```
custom/views/partials/footer.hbs
```

## Usage with Docker

<Steps>
  <Step title="Ensure the custom volume is accessible">
    If you're using the official Docker image, make sure `/kutt/custom` is mounted as a volume. The default `docker-compose.yml` already includes this mount.

    If you built the image locally, the `custom/` folder is already included in the image.
  </Step>
  <Step title="Copy your files into the container">
    Use `docker cp` to copy your local `custom/` folder into the running container:

    ```bash
    docker cp custom kutt-server-1:/kutt
    ```

    Replace `kutt-server-1` with your actual container name.
  </Step>
  <Step title="Restart the Kutt container">
    Changes take effect after a restart:

    ```bash
    docker restart kutt-server-1
    ```
  </Step>
</Steps>
